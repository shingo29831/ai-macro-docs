5. データ要件と構造定義

本システムで扱うデータ定義です。記録時に各モジュール（YOLO、OCR、Python）が個別に取得する「生データ」の構造を先に行い、それらをコンテキスト理解や自己修復のために集約・紐付けした「統合データ」の構造を後ろに定義します。

5.1. 生データ構造定義

解析モード（録画）の開始時、各認識エンジンおよびフックモジュールから独立して出力・保存されるデータ構造です。

5.1.1. UI認識データ（YOLO Engine）

プロパティ (階層) 型 説明・格納される値
uiAnalysis.timestamp number 画像を取得・解析した時間（Unixタイムスタンプ）
uiAnalysis.boundingBox object 認識したUI要素のバウンディングボックス座標 {x, y, width, height}
uiAnalysis.type string 物体認識されたUIの種類（"button", "dropdown", "input" など）
uiAnalysis.confidence number AIによるUI認識の信頼度・精度 (0.0〜1.0)

5.1.2. テキスト認識データ（OCR Engine）

プロパティ (階層) 型 説明・格納される値
textAnalysis.timestamp number 画像を取得・解析した時間（Unixタイムスタンプ）
textAnalysis.boundingBox object 認識したテキスト領域のバウンディングボックス座標 {x, y, width, height}
textAnalysis.content string OCRエンジンにより読み取られた文字列内容
textAnalysis.confidence number OCRによるテキスト認識の信頼度・精度 (0.0〜1.0)

5.1.3. 入力操作データ（Python OS Hook）

プロパティ (階層) 型 説明・格納される値
inputLog.timestamp number OSレベルで操作をフック・検出した時間（Unixタイムスタンプ）
inputLog.type string 入力操作の種類（"click_down", "key_down" など）
inputLog.content string 具体的な入力内容（"left_click", "Enter", または入力文字など）
inputLog.croppedImagePath string 操作が行われた位置の周辺を切り抜いた画像ファイルのローカルパス
inputLog.windowName string 操作対象となったアプリケーションのウィンドウタイトル名
inputLog.windowSize object 対象ウィンドウの全体サイズ {width, height}
inputLog.windowCoordinates object 対象ウィンドウのデスクトップ上における絶対座標 {x, y}
inputLog.cursorCoordinates object 操作が実行された瞬間のマウスカーソルの絶対座標 {x, y}

5.2. 生データ JSONフォーマット例

各検出モジュールが単独でディスクまたはメモリ上に保存する段階のJSON表現です。

5.2.1. UI認識データ例（YOLO Engine）

{
  "uiAnalysis": {
    "timestamp": 1717654800100,
    "boundingBox": { "x": 1220, "y": 280, "width": 60, "height": 40 },
    "type": "button",
    "confidence": 0.95
  }
}

5.2.2. テキスト認識データ例（OCR Engine）

{
  "textAnalysis": {
    "timestamp": 1717654800105,
    "boundingBox": { "x": 1225, "y": 285, "width": 50, "height": 30 },
    "content": "保存する",
    "confidence": 0.98
  }
}

5.2.3. 入力操作データ例（Python OS Hook）

{
  "inputLog": {
    "timestamp": 1717654800123,
    "type": "click_down",
    "content": "left_click",
    "croppedImagePath": "/logs/session_1/step_012_crop.png",
    "windowName": "社内システム - Google Chrome",
    "windowSize": { "width": 1920, "height": 1080 },
    "windowCoordinates": { "x": 0, "y": 0 },
    "cursorCoordinates": { "x": 1250, "y": 300 }
  }
}

5.3. 統合データ構造定義

各自律機能やAI生成コアが「コンテキスト理解」および「相対座標によるウィンドウ移動耐性」を確保するため、上記の生データをタイムスタンプや座標情報を基準に結合・構造化したデータモデルです。

プロパティ (階層) 型 説明・格納される値
id string 統合ログの一意なID (例: "step_001")
action.type enum 操作種別 ("click", "double_click", "keypress")
action.coordinates object 画面全体の絶対座標 {x, y}
action.uiImageBase64 string クリックしたUI（カーソル周辺アイコン）のBase64画像文字列。修復時のテンプレートマッチングに使用
action.keyContent string 入力されたキーの内容 (例: "Enter", "Hello") ※keypress時のみ
timestamp number アクション取得時のUnixタイムスタンプ。次アクションまでの待機時間算出に使用
context.screenshotBeforePath string 操作前の全体画面スクリーンショットのローカルパス
context.screenshotAfterPath string 操作後の全体画面スクリーンショットのローカルパス
context.activeWindowTitle string 操作対象となったアクティブウィンドウのタイトル名
context.relativeCoordinates object アクティブウィンドウの左上を原点(0,0)とした相対座標 {x, y}。ウィンドウ移動時の追従に使用
visionAnalysis.uiType string YOLO等が判定したUIの種類 ("button", "link", "input" 等)。AI動的再生成の基準に使用
visionAnalysis.uiDetails.boundingBox object UI要素の境界ボックス {x, y, width, height}
visionAnalysis.uiDetails.text string OCRエンジンによって読み取られたUI内のテキスト ("ログイン", "保存" 等)

{
  "id": "step_012",
  "rawLog": {
    "action": {
      "type": "click",
      "coordinates": { "x": 1250, "y": 300 },
      "uiImageBase64": "iVBORw0KGgoAAAANSUhEUgAAACAAAAAgCAYAAABzenr0...",
      "keyContent": null
    },
    "timestamp": 1717654800123,
    "context": {
      "screenshotBeforePath": "/logs/session_1/step_012_before.png",
      "screenshotAfterPath": "/logs/session_1/step_012_after.png",
