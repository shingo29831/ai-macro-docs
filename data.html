5. データ要件と構造定義

本システムで扱うデータ定義です。記録時に各モジュール（YOLO、OCR、Python）が個別に取得する「生データ」の構造を先に行い、それらをコンテキスト理解や自己修復のために集約・紐付けした「統合データ」の構造を後ろに定義します。

5.1. 生データ構造定義

解析モード（録画）の開始時、各認識エンジンおよびフックモジュールから独立して出力・保存されるデータ構造です。

5.1.1. UI認識データ（YOLO Engine）

プロパティ (階層) 型 説明・格納される値
uiAnalysis.timestamp number 画像を取得・解析した時間（Unixタイムスタンプ）
uiAnalysis.boundingBox object 認識したUI要素のバウンディングボックス座標 {x, y, width, height}
uiAnalysis.type string 物体認識されたUIの種類（"button", "dropdown", "input" など）
uiAnalysis.confidence number AIによるUI認識の信頼度・精度 (0.0〜1.0)

5.1.2. テキスト認識データ（OCR Engine）

プロパティ (階層) 型 説明・格納される値
textAnalysis.timestamp number 画像を取得・解析した時間（Unixタイムスタンプ）
textAnalysis.boundingBox object 認識したテキスト領域のバウンディングボックス座標 {x, y, width, height}
textAnalysis.content string OCRエンジンにより読み取られた文字列内容
textAnalysis.confidence number OCRによるテキスト認識の信頼度・精度 (0.0〜1.0)

5.1.3. 入力操作データ（Python OS Hook）

プロパティ (階層) 型 説明・格納される値
inputLog.timestamp number OSレベルで操作をフック・検出した時間（Unixタイムスタンプ）
inputLog.type string 入力操作の種類（"click_down", "key_down" など）
inputLog.content string 具体的な入力内容（"left_click", "Enter", または入力文字など）
inputLog.croppedImagePath string 操作が行われた位置の周辺を切り抜いた画像ファイルのローカルパス
inputLog.windowName string 操作対象となったアプリケーションのウィンドウタイトル名
inputLog.windowSize object 対象ウィンドウの全体サイズ {width, height}
inputLog.windowCoordinates object 対象ウィンドウのデスクトップ上における絶対座標 {x, y}
inputLog.cursorCoordinates object 操作が実行された瞬間のマウスカーソルの絶対座標 {x, y}

5.2. 生データ JSONフォーマット例

各検出モジュールが単独でディスクまたはメモリ上に保存する段階のJSON表現です。

5.2.1. UI認識データ例（YOLO Engine）

{
  "uiAnalysis": {
    "timestamp": 1717654800100,
    "boundingBox": { "x": 1220, "y": 280, "width": 60, "height": 40 },
    "type": "button",
    "confidence": 0.95
  }
}

5.2.2. テキスト認識データ例（OCR Engine）

{
  "textAnalysis": {
    "timestamp": 1717654800105,
    "boundingBox": { "x": 1225, "y": 285, "width": 50, "height": 30 },
    "content": "保存する",
    "confidence": 0.98
  }
}

5.2.3. 入力操作データ例（Python OS Hook）

{
  "inputLog": {
    "timestamp": 1717654800123,
    "type": "click_down",
    "content": "left_click",
    "croppedImagePath": "/logs/session_1/step_012_crop.png",
    "windowName": "社内システム - Google Chrome",
    "windowSize": { "width": 1920, "height": 1080 },
    "windowCoordinates": { "x": 0, "y": 0 },
    "cursorCoordinates": { "x": 1250, "y": 300 }
  }
}

5.3. 統合データ構造定義

自律機能やAI生成コアが「コンテキスト理解」を確保するため、生データをウィンドウ基準の階層型ツリー（DOMライク）に統合したデータモデルです。

プロパティ (階層) 型 説明・格納される値
id string 統合ログの一意なID (例: "step_001")
timestamp number アクション実行時のUnixタイムスタンプ
screenshotBeforePath / AfterPath string 操作前・操作後の全体画面スクリーンショットのローカルパス
window.name string 操作対象となったアクティブウィンドウの名前（タイトル）
window.size object ウィンドウのサイズ {width, height}
window.coordinates object ウィンドウの画面上の絶対座標 {x, y}
window.UIs array 対象ウィンドウ内で認識され、操作に関与したUI要素の配列
window.UIs[].type string UIの種類（"button", "input" 等）
window.UIs[].relativeBoundingBox object 対象ウィンドウの左上を原点(0,0)とした相対座標とサイズ {x, y, width, height}
window.UIs[].confidence number AIによるUI認識精度 (0.0〜1.0)
window.UIs[].action object 当該UIに対して行われた入力操作の詳細
window.UIs[].action.inputType string 入力タイプ（"click_down", "key_down" 等）
window.UIs[].action.inputValue string 入力内容（"left_click", "Enter" 等）
window.UIs[].action.cursorRelativeCoordinates object 対象ウィンドウ内でのカーソル相対座標 {x, y}
window.UIs[].action.diffRatio number 操作による前画面変化率（待機判定に使用）
window.UIs[].action.croppedImagePath string 操作周辺の切り抜き画像パス
window.UIs[].context array UI要素を補足する周辺情報（テキストやアイコン等）の配列
window.UIs[].context[].type string コンテキストの種類（"text", "icon" 等）
window.UIs[].context[].content string 読み取られたテキストや記号の種類名
window.UIs[].context[].relativeBoundingBox object 対象ウィンドウ内での相対座標とサイズ {x, y, width, height}
window.UIs[].context[].confidence number 認識精度 (0.0〜1.0)
window.UIs[].context[].parentRelevance number 親要素（対象UI）との関連度合い（距離や意味合いから算出）

5.4. 統合データ JSONフォーマット例

{
  "id": "step_012",
  "timestamp": 1717654800123,
  "screenshotBeforePath": "/logs/session_1/step_012_before.png",
  "screenshotAfterPath": "/logs/session_1/step_012_after.png",
  "window": {
    "name": "社内システム - Google Chrome",
    "size": { "width": 1920, "height": 1080 },
    "coordinates": { "x": 0, "y": 0 },
    "UIs": [
      {
        "type": "button",
        "relativeBoundingBox": { "x": 1220, "y": 280, "width": 60, "height": 40 },
        "confidence": 0.95,
        "action": {
          "inputType": "click_down",
          "inputValue": "left_click",
          "cursorRelativeCoordinates": { "x": 1250, "y": 300 },
          "diffRatio": 0.12,
          "croppedImagePath": "/logs/session_1/step_012_crop.png"
        },
        "context": [
          {
            "type": "text",
            "content": "保存する",
            "relativeBoundingBox": { "x": 1225, "y": 285, "width": 50, "height": 30 },
            "confidence": 0.98,
            "parentRelevance": 0.99
          },
          {
            "type": "icon",
            "content": "save_icon",

5.5. ワークフローデータ構造定義

統合ログから抽出・生成され、実行エンジンへと引き渡されるマクロシナリオの構造定義です。

プロパティ (階層) 型 説明・格納される値
workflow_ID string ワークフロー（マクロ）を一意に識別するID
target_ID string 操作対象となる主要なアプリケーションの識別子ID
events array ワークフローを構成する一連の操作イベントの配列
events[].event_id string 各操作イベントの一意なID
events[].timestamp number イベントが検出または生成されたUnixタイムスタンプ
events[].action object 実行された具体的なユーザー操作内容の詳細
events[].action.type string 入力操作の種類（"click", "key_down", "text_input" など）
events[].action.button string 使用されたマウスボタン（"left", "right" など、キー入力時は省略可）
events[].action.modifiers array 同時に押下された修飾キー（"ctrl", "shift", "alt" など）の配列
events[].context object 操作実行時における対象要素および画面の状態コンテキスト
events[].context.interacted_element object ユーザーが実際に操作を加えたUI要素の静的・意味的情報
events[].context.interacted_element.element_id string 操作対象となったUI要素を特定する内部ID
events[].context.interacted_element.ui_type string 要素のオブジェクトタイプ（"button", "input", "checkbox" 等）
events[].context.interacted_element.semantic_role string 要素の持つ意味的役割（"submit", "cancel", "search_box" 等）
events[].context.interacted_element.location_context string 配置上の視覚的コンテキスト（"bottom_right", "top_nav" 等）
events[].context.pre_action_state object アクションが実行される直前のシステム・画面ステータス
events[].context.pre_action_state.image_ref string 自己修復や画像差分チェックに使用する操作前基準スクショのローカルパス

5.6. ワークフローデータ JSONフォーマット例

構造定義に基づき、生成コアから出力されるワークフローデータのJSON表現です。

{
  "workflow_ID": "wf_12345",
  "target_ID": "target_app_001",
  "events": [
    {
      "event_id": "evt_001",
      "timestamp": 1717654800123,
      "action": {
        "type": "click",
        "button": "left",
        "modifiers": ["shift"]
      },
      "context": {
        "interacted_element": {
          "element_id": "btn_save",
          "ui_type": "button",
          "semantic_role": "submit",
          "location_context": "bottom_right"
        },
        "pre_action_state": {
          "image_ref": "/logs/wf_12345/evt_001_pre.png"
        }
      }
    }
  ]
}
